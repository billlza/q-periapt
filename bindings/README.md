# Language bindings

One Rust core (`pqt-ffi`, the C ABI), three faces: **C**, **Swift**, **Kotlin**.
The headline property is *byte-identical results across platforms* — so all
bindings are validated against one shared oracle.

## Cross-platform consistency oracle

[`shared-test-vectors.json`](shared-test-vectors.json) is generated from the Rust
core:

```sh
cargo run -p pqt-backends --example refvec > bindings/shared-test-vectors.json
```

It contains a full ContextBound vector (keys, randomness, ciphertexts, and the
resulting 32-byte `secret`). Every binding's test suite must:

1. `decapsulate(...)` the vector's inputs and assert the result equals `secret`;
2. `encapsulate(...)` with the vector's randomness and assert it reproduces
   `ct_pq`, `ct_trad`, and `secret`.

If any platform disagrees by a single byte, the suites fail — that is the
cross-platform guarantee, made executable.

## Artifacts

- C header: [`../crates/pqt-ffi/include/pqt.h`](../crates/pqt-ffi/include/pqt.h)
  (generated by cbindgen; regenerate with the command in its banner).
- Static/shared lib: `cargo build -p pqt-ffi --release` →
  `target/release/libpqt_ffi.{a,dylib,so}`.

## Status

| Face | Consistency vs shared vector | How |
|---|---|---|
| **Rust core** | ✅ source of truth | `cargo test` (combiner KATs, X-Wing KAT) |
| **C ABI** (`pqt-ffi`) | ✅ verified | `cargo test -p pqt-ffi` (`ffi_matches_shared_vector`) |
| **Swift** (`swift/`) | ✅ verified | `swift test` (CI: `bindings-swift` on macOS) |
| **WASM** (`pqt-wasm`) | ✅ logic verified + builds wasm32 | `cargo test -p pqt-wasm`; CI builds `wasm32` |
| **Kotlin** (`kotlin/`) | ✅ verified | `gradle test` (Panama FFM, JDK ≥22); CI: `bindings-kotlin` |

**All five faces decapsulate `shared-test-vectors.json` to the same 32-byte secret,
byte-for-byte** — verified locally (Rust/C/Swift/WASM-on-Node/Kotlin) and gated in
CI. Kotlin needs a JDK ≥22 (stable FFM):

```sh
cargo build -p pqt-ffi --release
JAVA_HOME=/path/to/jdk22+ gradle -p bindings/kotlin test
```
